# Security Policy

## 지원 버전

현재 `main` 브랜치만 보안 업데이트를 받습니다. 별도 LTS 브랜치는 없습니다.

## 취약점 신고

보안 취약점을 발견하셨다면 **공개 이슈로 등록하지 말아 주세요**.

다음 방법 중 하나로 비공개 채널을 통해 알려주세요:

1. **GitHub Security Advisory (권장)** — 저장소의 `Security` 탭 → `Report a vulnerability`
2. **이메일** — hong-ik 메인테이너에게 직접 (저장소 README의 연락처 또는 커밋 작성자 이메일)

## 신고 시 포함하면 도움 되는 정보

- 취약점 유형 (예: 샌드박스 우회, 인터프리터 RCE, 메모리 안전성 등)
- 영향받는 파일/함수/버전
- 재현 가능한 PoC 코드 또는 입력
- 잠재적 영향 범위 (정보 누출 / 서비스 거부 / 코드 실행 등)
- 가능한 완화책 제안 (선택)

## 응답 일정

- **48시간 이내**: 신고 접수 확인
- **7일 이내**: 영향도 평가 + 수정 일정 공유
- **30일 이내**: 패치 및 공개 일정 합의 (심각도에 따라 단축)

## 책임 있는 공개

수정 패치가 배포된 후 신고자의 허락을 받아 다음 정보를 공개합니다:

- 취약점 요약 (CVE-스타일 식별자 부여 가능)
- 신고자 크레딧 (원치 않으면 익명 처리)
- 영향받은 버전과 수정된 버전

확정된 수정 전에는 신고자도 비공개를 유지해 주시기 바랍니다.

## 알려진 보안 고려 사항

`hong-ik`은 임의 사용자 코드를 실행하는 인터프리터입니다. 다음은 의도된 보호 장치이며, 우회 사례를 찾으시면 위 절차로 신고해 주세요:

- `가져오기`는 `..`, 절대 경로, 드라이브 문자 차단 (CWD 하위만 허용)
- `파일읽기`/`파일쓰기` 동일 정책
- VM 호출 프레임 스택 underflow 시 명시적 예외 (UB 방지)
- 정수 오버플로 시 double로 폴백
